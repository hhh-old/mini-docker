# 宿主机与容器文件系统映射关系

本文档详细解释 mini-docker 中宿主机文件系统和容器文件系统的结构以及它们之间的映射关系。

## 1. 宿主机文件系统结构

```
宿主机 /
├── bin/
├── etc/
├── var/
│   ├── run/
│   │   └── mini-docker/                    ← 容器元数据存储
│   │       ├── <container-id-1>.json       ← 容器1的信息
│   │       └── <container-id-2>.json       ← 容器2的信息
│   │
│   └── lib/
│       └── mini-docker/
│           ├── images/                     ← 镜像存储
│           │   └── myos/
│           │       ├── metadata.json       ← 镜像元数据
│           │       └── rootfs/             ← 镜像的根文件系统（只读）
│           │           ├── bin/
│           │           │   ├── busybox
│           │           │   ├── sh -> busybox
│           │           │   ├── ls -> busybox
│           │           │   └── ... (210+ 命令)
│           │           ├── etc/
│           │           ├── lib/
│           │           └── ...
│           │
│           └── containers/                 ← 容器数据存储
│               └── <container-id>/         ← 容器的可写层
│                   └── overlay/
│                       ├── merged/         ← 合并后的视图
│                       ├── upper/          ← 可写层（容器修改的内容）
│                       └── work/           ← OverlayFS 工作目录
│
├── proc/                                   ← 宿主机的进程信息
└── tmp/                                    ← 宿主机的临时文件
```

## 2. 容器内文件系统结构

```
容器内 / (实际是 merged/ 目录)
├── bin/           ← 来自 lowerdir (镜像)
│   ├── busybox
│   ├── sh -> busybox
│   └── ...
├── etc/           ← 来自 lowerdir (镜像)
│   ├── hostname
│   ├── resolv.conf
│   └── ...
├── lib/           ← 来自 lowerdir (镜像)
├── proc/          ← 虚拟文件系统 (内核生成)
│   ├── cpuinfo
│   ├── meminfo
│   └── <pid>/     ← 只显示容器内的进程
├── tmp/           ← tmpfs (内存)
└── ...
```

## 3. 映射关系图

```
宿主机路径                                        容器内路径
─────────────────────────────────────────────────────────────

/var/lib/mini-docker/images/myos/rootfs/    →    / (只读部分)
├── bin/busybox                             →    /bin/busybox
├── etc/hostname                            →    /etc/hostname
└── ...                                     →    ...

/var/lib/mini-docker/containers/<id>/overlay/
├── upper/                                  →    / (可写部分)
│   ├── tmp/xxx                             →    /tmp/xxx
│   └── root/xxx                            →    /root/xxx
├── merged/                                 →    / (整个根目录)
│   (合并 lowerdir + upper 的视图)
└── work/                                   →    (OverlayFS 内部使用)

内核虚拟                                      →    /proc
(无对应宿主机路径)                             →    (容器看到自己namespace的进程)

内存                                          →    /tmp
(无对应宿主机路径)                             →    (独立的内存空间)
```

## 4. OverlayFS 合并机制

### 4.1 三层结构

```
lowerdir (镜像)           upper (可写层)          merged (容器看到的)
─────────────────────────────────────────────────────────────────────

/bin/                     /bin/                   /bin/
├── busybox               ├── myapp               ├── busybox
├── sh → busybox          └── myscript            ├── sh → busybox
└── ls → busybox                                  ├── ls → busybox
                                                  ├── myapp        ← 来自 upper
                                                  └── myscript     ← 来自 upper

/etc/                     /etc/                   /etc/
├── hostname              └── myconfig.conf       ├── hostname      ← 来自 lower
└── resolv.conf                                   ├── resolv.conf   ← 来自 lower
                                                  └── myconfig.conf ← 来自 upper
```

### 4.2 读写规则

| 操作 | 来源 | 说明 |
|------|------|------|
| 读取文件 | 先查 upper，再查 lowerdir | upper 优先级更高 |
| 修改文件 | 复制到 upper，在 upper 修改 | 写时复制 (Copy-on-Write) |
| 创建文件 | 直接写入 upper | 新文件存在可写层 |
| 删除文件 | 在 upper 创建 whiteout 标记 | 标记文件已删除 |

### 4.3 代码实现

```go
// 创建 OverlayFS 目录
func createOverlayDirs(containerID string) (*OverlayDirs, error) {
```

## 5. 完整映射流程

### 5.1 流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                         宿主机视角                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  /var/lib/mini-docker/images/myos/rootfs/   (镜像，只读)         │
│         ↓                                                       │
│  /var/lib/mini-docker/containers/<id>/overlay/                   │
│  ├── lowerdir = /var/lib/mini-docker/images/myos/rootfs/        │
│  ├── upperdir = /var/lib/mini-docker/containers/<id>/overlay/upper/  │
│  ├── workdir  = /var/lib/mini-docker/containers/<id>/overlay/work/   │
│  └── merged   = /var/lib/mini-docker/containers/<id>/overlay/merged/ │
│         ↓                                                       │
│  pivot_root 切换根目录                                           │
│         ↓                                                       │
└─────────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────────┐
│                         容器视角                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  / (根目录)                                                      │
│  ├── bin/        ← 来自 lowerdir (镜像)                         │
│  ├── etc/        ← 来自 lowerdir (镜像) + upper (修改)          │
│  ├── proc/       ← 虚拟文件系统 (内核生成，namespace 隔离)       │
│  ├── tmp/        ← tmpfs (内存，独立空间)                        │
│  └── ...                                                        │
│                                                                 │
│  容器进程看到的路径都是相对于新的根目录 /                         │
│  无法访问宿主机的 /var/lib/mini-docker/...                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 代码流程

```go
func SetupRootFS(rootFSPath string, overlay *OverlayDirs) error {
    // 1. 重新挂载根目录为私有（切断与宿主机的联系）
    unix.Mount("", "/", "", unix.MS_PRIVATE|unix.MS_REC, "")

    // 2. 挂载 OverlayFS（合并镜像和可写层）
    options := fmt.Sprintf("lowerdir=%s,upperdir=%s,workdir=%s",
        rootFSPath, overlay.Upper, overlay.Work)
    unix.Mount("overlay", overlay.Merged, "overlay", 0, options)

    // 3. 切换根目录（隐藏宿主机路径）
    unix.PivotRoot(overlay.Merged, pivotDir)
    unix.Chdir("/")

    // 4. 挂载虚拟文件系统
    unix.Mount("proc", "/proc", "proc", 0, "")
    unix.Mount("tmpfs", "/tmp", "tmpfs", unix.MS_NOSUID|unix.MS_NODEV, "size=64m")

    return nil
}
```

## 6. 关键点总结

### 6.1 映射关系表

| 宿主机路径 | 容器路径 | 类型 | 说明 |
|-----------|---------|------|------|
| `/var/lib/mini-docker/images/myos/rootfs/` | `/` | 只读 | 镜像内容 |
| `/var/lib/mini-docker/containers/<id>/overlay/upper/` | `/` | 可写 | 容器修改 |
| `/var/lib/mini-docker/containers/<id>/overlay/merged/` | `/` | 合并 | 容器看到的完整视图 |
| (内核虚拟) | `/proc` | 虚拟 | 进程信息，namespace 隔离 |
| (内存) | `/tmp` | 内存 | 临时文件，容器独立 |

### 6.2 文件系统类型对比

| 文件系统 | 数据来源 | 容器路径 | 特点 |
|---------|---------|---------|------|
| **overlay** | 宿主机磁盘 | `/` | 只读镜像 + 可写层 |
| **proc** | 内核虚拟 | `/proc` | 实时生成，namespace 隔离 |
| **tmpfs** | 内存 | `/tmp` | 容器停止后清除 |

### 6.3 隔离机制

| 机制 | 作用 | 代码位置 |
|------|------|----------|
| **OverlayFS** | 合并只读镜像和可写层 | `rootfs_linux.go` |
| **pivot_root** | 切换根目录，隐藏宿主机路径 | `rootfs_linux.go` |
| **Mount Namespace** | 隔离文件系统视图 | `namespace_linux.go` |
| **proc 虚拟文件系统** | 隔离进程信息 | `rootfs_linux.go` |
| **tmpfs** | 隔离临时文件 | `rootfs_linux.go` |

## 7. 实际示例

### 7.1 创建容器前

```bash
# 宿主机上的目录结构
$ ls /var/lib/mini-docker/images/myos/rootfs/
bin  etc  lib  proc  tmp  usr  ...

$ ls /var/lib/mini-docker/containers/
(empty)
```

### 7.2 创建容器后

```bash
# 宿主机上的目录结构
$ ls /var/lib/mini-docker/containers/<id>/overlay/
merged  upper  work

$ ls /var/lib/mini-docker/containers/<id>/overlay/upper/
(empty - 容器还没修改任何文件)

$ ls /var/lib/mini-docker/containers/<id>/overlay/merged/
bin  etc  lib  proc  tmp  usr  ...  (合并后的完整文件系统)
```

### 7.3 容器内视角

```bash
# 容器内看到的
$ ls /
bin  etc  lib  proc  tmp  usr  ...

$ cat /etc/hostname
mini-docker

$ ls /proc
1  cpuinfo  meminfo  ...  (只有容器内的进程)

$ echo "hello" > /tmp/test.txt
$ ls /var/lib/mini-docker/containers/<id>/overlay/upper/tmp/
test.txt  (文件实际存在宿主机的 upper 目录)
```

## 8. 总结

1. **宿主机路径 → 容器路径**：通过 OverlayFS + pivot_root 实现映射
2. **只读层 (lowerdir)**：镜像内容，所有容器共享
3. **可写层 (upper)**：每个容器独立，存储修改和新增文件
4. **虚拟文件系统**：proc、tmpfs 由内核/内存提供，与宿主机隔离
5. **namespace 隔离**：确保容器只能看到自己的进程、网络、文件系统

这种设计实现了：
- **资源高效**：多个容器共享同一个镜像
- **隔离安全**：容器之间互不影响
- **快速启动**：只需创建可写层，无需复制整个文件系统
